Part 1: RANK vs DENSE_RANK vs ROW_NUMBER

These window functions assign sequence numbers to rows. They differ in handling ties.

Quick Comparison

Scenario ROW_NUMBER RANK DENSE_RANK
Handles Ties Every row gets unique number Ties get same rank, gaps after Ties get same rank, no gaps
Use Case Pagination, unique identification Competition scoring Continuous ranking
Example 1, 2, 3, 4, 5 1, 2, 2, 4, 5 1, 2, 2, 3, 4

Template (Basic Syntax)

SELECT 
    column_name,
    ROW_NUMBER() OVER (ORDER BY column_name) AS row_num,
    RANK() OVER (ORDER BY column_name) AS rank_value,
    DENSE_RANK() OVER (ORDER BY column_name) AS dense_rank_value
FROM table_name;

Real Example: Trade Type Analysis

;WITH TradeStats AS (
    SELECT 
        TradeType,
        COUNT(*) AS FrequencyCount
    FROM [repo].[tbltradetype]
    GROUP BY TradeType
)
SELECT 
    TradeType,
    FrequencyCount,
    -- ROW_NUMBER: Unique number for each row (1, 2, 3...)
    ROW_NUMBER() OVER (ORDER BY FrequencyCount DESC) AS rn_unique,
    
    -- RANK: Skips numbers after ties (1, 2, 2, 4...)
    RANK() OVER (ORDER BY FrequencyCount DESC) AS rank_with_gap,
    
    -- DENSE_RANK: No gaps (1, 2, 2, 3...)
    DENSE_RANK() OVER (ORDER BY FrequencyCount DESC) AS dense_rank_no_gap
FROM TradeStats
ORDER BY FrequencyCount DESC;
💡 Pro Tip 1: PARTITION BY
Use PARTITION BY to group before ranking: 
ROW_NUMBER() OVER (PARTITION BY TradeType ORDER BY TradeID) AS seq_per_trade
💡 Pro Tip 2: Top N Per Group
Get top 3 items per trade type: 
;WITH RankedTrades AS (
    SELECT *, ROW_NUMBER() OVER (PARTITION BY TradeType ORDER BY TradeID DESC) AS rn
    FROM [repo].[tbltradetype]
)
SELECT * FROM RankedTrades WHERE rn <= 3;

Part 2: Temp Table vs Table Variable vs CTE

Quick Comparison

Feature #Temp Table @Table Variable CTE
Scope Session-level (multiple batches) Batch-level (single batch only) Single query only
Persistence Physically stored in tempdb In-memory (mostly) Virtual (doesn't persist)
Indexes Can create indexes No indexes allowed No indexes
Performance Good for large datasets Good for small datasets Best for readable queries
Best For Multi-step processing Single-batch calculations Recursive queries, readability

Template 1: Temp Table

CREATE TABLE #temp_table_name (
    column_name data_type,
    column_name data_type
);

INSERT INTO #temp_table_name SELECT ...
SELECT * FROM #temp_table_name;

DROP TABLE #temp_table_name;  -- Auto-drops when session ends

Example 1: Temp Table - Multi-Step Analysis

-- STEP 1: Create temp table
CREATE TABLE #TradeTypeStats (
    TradeType NVARCHAR(50),
    TradeCount INT,
    ProcessedDate DATETIME DEFAULT GETDATE()
);

-- STEP 2: Populate
INSERT INTO #TradeTypeStats (TradeType, TradeCount)
SELECT TradeType, COUNT(*) FROM [repo].[tbltradetype]
GROUP BY TradeType;

-- STEP 3: Use multiple times
SELECT * FROM #TradeTypeStats WHERE TradeCount > 10;
SELECT TOP 3 * FROM #TradeTypeStats ORDER BY TradeCount DESC;
SELECT SUM(TradeCount) AS TotalTrades FROM #TradeTypeStats;

DROP TABLE #TradeTypeStats;

Template 2: Table Variable

DECLARE @table_variable_name TABLE (
    column_name data_type,
    column_name data_type
);

INSERT INTO @table_variable_name SELECT ...
SELECT * FROM @table_variable_name;

Example 2: Table Variable - Single Batch

DECLARE @TradeStats TABLE (
    TradeType NVARCHAR(50),
    TradeCount INT
);

INSERT INTO @TradeStats (TradeType, TradeCount)
SELECT TradeType, COUNT(*) 
FROM [repo].[tbltradetype]
GROUP BY TradeType;

SELECT * FROM @TradeStats;
SELECT AVG(TradeCount) AS AvgCount FROM @TradeStats;

Template 3: CTE (Common Table Expression)

WITH cte_name AS (
    SELECT column_name FROM table_name WHERE condition
)
SELECT * FROM cte_name;

Example 3: CTE - Readable Multi-Step Query

WITH TradeTypeStats AS (
    -- Step 1: Calculate stats
    SELECT 
        TradeType,
        COUNT(*) AS TradeCount,
        MAX(TradeID) AS MaxTradeID
    FROM [repo].[tbltradetype]
    GROUP BY TradeType
),
FilteredStats AS (
    -- Step 2: Filter
    SELECT * FROM TradeTypeStats WHERE TradeCount > 0
),
FinalResults AS (
    -- Step 3: Calculate percentage
    SELECT 
        TradeType,
        TradeCount,
        ROUND(CAST(TradeCount AS FLOAT) / (SELECT SUM(TradeCount) FROM FilteredStats) * 100, 2) AS PercentageOfTotal
    FROM FilteredStats
)
SELECT * FROM FinalResults ORDER BY TradeCount DESC;
When to Use What:
  • Use #Temp Table for large datasets processed in multiple steps
  • Use @Table Variable for small lookup data in a single batch
  • Use CTE to make complex queries readable

Part 3: Join Types Explained

Join Types Overview

Join Type Returns Use When
INNER JOIN Only matching rows from both tables You need data that exists in BOTH tables
LEFT OUTER JOIN All from LEFT + matching from RIGHT You want all from left, even if no match
RIGHT OUTER JOIN Matching from LEFT + all from RIGHT You want all from right, even if no match
FULL OUTER JOIN All rows from BOTH tables You want everything, matched or not
SELF JOIN Table joined to itself Comparing rows within same table
CROSS JOIN Cartesian product Every row from A with every row from B

Template 1: INNER JOIN

SELECT 
    table_a.column_name,
    table_b.column_name
FROM table_a
INNER JOIN table_b ON table_a.key = table_b.key;

Example 1: INNER JOIN

-- Only trades with matching trade type
SELECT 
    t.TradeID,
    t.TradeType,
    tt.TypeCode
FROM [repo].[tbltradetype] t
INNER JOIN [repo].[tbltradetype] tt ON t.TradeType = tt.TradeType;

Template 2: LEFT OUTER JOIN

SELECT 
    table_a.column_name,
    table_b.column_name
FROM table_a
LEFT OUTER JOIN table_b ON table_a.key = table_b.key;

Example 2: LEFT OUTER JOIN

-- All trades from left, even if no match
SELECT 
    t.TradeID,
    t.TradeType,
    t.TypeCode,
    ISNULL(tt.TypeCode, 'NO_MATCH') AS MatchedCode
FROM [repo].[tbltradetype] t
LEFT OUTER JOIN [repo].[tbltradetype] tt ON t.TradeType = tt.TradeType 
    AND tt.Active = 1;

Template 3: FULL OUTER JOIN

SELECT 
    table_a.column_name,
    table_b.column_name
FROM table_a
FULL OUTER JOIN table_b ON table_a.key = table_b.key;

Example 3: FULL OUTER JOIN

-- All rows from both tables
SELECT 
    COALESCE(t.TradeID, tt.TradeID) AS TradeID,
    t.TradeType AS LeftTradeType,
    tt.TradeType AS RightTradeType
FROM [repo].[tbltradetype] t
FULL OUTER JOIN [repo].[tbltradetype] tt ON t.TradeType = tt.TradeType;

Template 4: SELF JOIN

SELECT 
    t1.column_name,
    t2.column_name
FROM table_name t1
INNER JOIN table_name t2 ON t1.key = t2.related_key;

Example 4: SELF JOIN

-- Find trades sharing same type code
SELECT 
    t1.TradeID AS TradeID_1,
    t1.TradeType AS TradeType_1,
    t2.TradeID AS TradeID_2,
    t2.TradeType AS TradeType_2
FROM [repo].[tbltradetype] t1
INNER JOIN [repo].[tbltradetype] t2 
    ON t1.TypeCode = t2.TypeCode 
    AND t1.TradeID < t2.TradeID;

Template 5: CROSS JOIN

SELECT 
    table_a.column_name,
    table_b.column_name
FROM table_a
CROSS JOIN table_b;

Example 5: CROSS JOIN

-- Create all combinations (3 types × 3 years = 9 rows)
DECLARE @TradeTypes TABLE (TradeType NVARCHAR(50));
INSERT INTO @TradeTypes VALUES ('FX'), ('EQ'), ('FI');

DECLARE @Years TABLE (YearValue INT);
INSERT INTO @Years VALUES (2024), (2025), (2026);

SELECT 
    tt.TradeType,
    y.YearValue,
    CONCAT(tt.TradeType, '_', y.YearValue) AS CombinationCode
FROM @TradeTypes tt
CROSS JOIN @Years y;
💡 Tip: NULL Handling in Outer Joins
Always check for NULLs to detect unmatched rows: 
WHERE joined_table.key IS NULL  -- Find unmatched rows

Part 4: Pro Tips & Tricks

Tip 1: Multiple Joins Chaining

SELECT t1.*, t2.*, t3.*
FROM Table1 t1
INNER JOIN Table2 t2 ON t1.ID = t2.ID
LEFT JOIN Table3 t3 ON t2.ID = t3.ID
INNER JOIN Table4 t4 ON t3.ID = t4.ID;

Tip 2: CTE vs Subquery

-- CTE (more readable, reusable)
WITH Stats AS (
    SELECT TradeType, COUNT(*) AS Cnt FROM [repo].[tbltradetype] GROUP BY TradeType
)
SELECT * FROM Stats WHERE Cnt > 1;

-- Subquery (older, harder to read)
SELECT * FROM (
    SELECT TradeType, COUNT(*) AS Cnt FROM [repo].[tbltradetype] GROUP BY TradeType
) AS Stats
WHERE Cnt > 1;

Tip 3: Recursive CTE for Hierarchies

WITH RecursiveHierarchy AS (
    -- Anchor: start with top-level
    SELECT ID, TradeType, 1 AS Level
    FROM [repo].[tbltradetype]
    WHERE TradeID = 1
    
    UNION ALL
    
    -- Recursive: find children
    SELECT t.ID, t.TradeType, r.Level + 1
    FROM [repo].[tbltradetype] t
    INNER JOIN RecursiveHierarchy r ON t.TradeID = r.ID
    WHERE r.Level < 10  -- Safety limit
)
SELECT * FROM RecursiveHierarchy;

Tip 4: Performance - Filter in JOIN Condition

-- BETTER: Filter in JOIN (more efficient)
SELECT *
FROM [repo].[tbltradetype] t1
INNER JOIN [repo].[tbltradetype] t2 ON t1.TradeID = t2.TradeID
    AND t1.Active = 1;  -- Filter during join, not after

Tip 5: Avoid N+1 Query Problem

-- GOOD: Single join query instead of loop
SELECT t.*, tt.*
FROM [repo].[tbltradetype] t
LEFT JOIN [repo].[tbltradetype] tt ON t.TradeType = tt.TradeType;
